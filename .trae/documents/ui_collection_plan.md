# UI 风格样板收集与代码生成计划

## 1. 目标概述
利用具备网页访问能力的 Agent，搜集国内外口碑极佳的大厂首页（如 Apple、Stripe、Vercel、Linear 等）及高质量 UI 设计组件网站，收集至少 100 个优秀的 UI 风格与组件。最终将这些收集到的 UI 转化为**前端代码组件**（默认基于 HTML/Tailwind CSS），并按“色系+风格+UI特征（如高圆角、大间距、玻璃拟物化等 tag）”进行分类和归档。

## 2. 架构与技术栈
*   **产物格式**：前端代码样板（HTML + Tailwind CSS，便于跨框架移植，或按需生成 React/Vue 组件）。
*   **分类系统**：基于 Tag（标签）的元数据管理系统。每个样板附带一个 JSON 配置文件或 Markdown Frontmatter，记录其特征。
*   **展示平台**：建立一个简单的本地索引页面（Gallery），支持按标签（色系、风格、UI特征）筛选和预览这 100+ 个样板。

---

## 3. 分阶段执行步骤

为了保证最终生成的 100 个样板符合您的审美要求，整个工程将采用**分阶段、需确认**的方式推进。

### 第一阶段：目标检索与清单收集 (Agent 探索期)
*   **任务描述**：Agent 自主访问互联网，检索并分析高质量的网页和 UI 库（如 Awwwards 获奖作品、Aceternity UI、Magic UI、大厂官网等）。
*   **具体行动**：
    1.  浏览并筛选至少 100 个高美观度的 UI 模块（如 Hero Section、卡片组、复杂表单、酷炫导航栏等）。
    2.  提取每个 UI 模块的设计特征，生成初步的 Tag（例如：`["深色模式", "科技风", "高圆角", "弥散阴影", "大间距"]`）。
    3.  将这 100 个候选清单汇总输出为一份文档（如 `ui_candidates_list.md` 或 `candidates.json`），包含：来源链接、UI 类型、初步提取的标签、简短描述。
*   **审查节点 1**：暂停执行，等待您（用户）对这 100 个候选清单及标签分类进行审查、删减或补充。确认无误后进入下一阶段。

### 第二阶段：工程初始化与分类体系搭建
*   **任务描述**：搭建本地样板库的代码基础框架。
*   **具体行动**：
    1.  初始化项目目录结构：
        ```text
        d:\code\UI-upgrade\
        ├── templates\           # 存放 100+ 个 UI 代码样板的目录
        │   ├── 001-stripe-card\ # 每个样板独立文件夹
        │   │   ├── index.html   # 样板代码
        │   │   └── meta.json    # 包含色系、风格、特征等 tags
        │   └── ...
        ├── gallery\             # 样板预览与筛选的本地展示页
        └── package.json         # (如有必要) 开发依赖
        ```
    2.  配置统一的 Tailwind CSS 环境，确保所有生成的样板在相同的样式基准下工作。

### 第三阶段：批量样板代码生成 (Agent 编码期)
*   **任务描述**：Agent 根据您确认的清单，逐一或分批生成对应的前端代码。
*   **具体行动**：
    1.  **分批执行**：为避免单次任务过载，Agent 将以每批次 10-20 个的速度进行代码生成。
    2.  **代码还原**：Agent 分析目标 UI 的截图或网页 DOM 结构，使用 HTML + Tailwind CSS 精确还原布局、间距、圆角、阴影、色彩等视觉特征。
    3.  **完善标签**：将确认后的 `色系`、`风格`、`UI特征` 写入每个样板的 `meta.json` 中。
*   **审查节点 2**：每完成一批（或全部完成后），您可以运行本地服务（如 Live Server 或 Vite）预览生成的样板，确保代码质量和还原度达到“高度美观协调”的标准。

### 第四阶段：整合与索引展示页构建
*   **任务描述**：制作一个方便您浏览和检索所有 100+ 样板的聚合页面。
*   **具体行动**：
    1.  读取所有样板的 `meta.json`。
    2.  生成一个类似于 UI 组件库官网的单页应用（Gallery），支持通过点击 Tag（如“大间距”、“极简风”、“蓝色系”）快速过滤展示样板。
    3.  提供一键复制代码的功能，方便后续直接在您的项目中使用。

---

## 4. 假设与约束
*   **审美对齐**：由于“美观度”具有一定主观性，第一阶段的清单确认至关重要，它将作为后续代码生成的基准。
*   **动效限制**：纯 CSS 动效会被保留，但高度复杂的 WebGL 或依赖特定 JS 库的复杂交互，在初版样板中可能会被简化为静态的完美视觉呈现。
*   **响应式设计**：默认所有样板均包含基础的移动端适配（Mobile-First）。

## 5. 验证方式
*   第一阶段完成时，输出的 `ui_candidates_list.md` 中需包含不少于 100 个有效条目及其详细 Tag。
*   最终阶段完成时，通过浏览器打开 `gallery/index.html`，能够顺畅浏览所有组件，筛选功能正常，且视觉呈现无样式崩塌。
