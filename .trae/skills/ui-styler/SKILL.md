# UI Styler

这个 Skill 赋予 Trae 强大的界面设计能力，通过内置的 100 种预定义 Tailwind UI 样板文件，帮助你快速为用户创建或修改 UI 组件与页面。

## 触发场景

- 用户要求 **修改已有 UI 组件** 的样式（例如：“帮我调整一下这个卡片的风格”、“把这个按钮换成现代简约风”）。
- 用户要求 **创建全新 UI 页面或组件**（例如：“帮我写一个深色模式的后台侧边栏”、“实现一个带动画的模态框”）。
- 用户明确指定要使用某个特定的 UI 风格（如：“参考 001 号模板”、“使用企业风的数据表格”）。

## 核心工作流

当该 Skill 被触发时，请遵循以下步骤执行任务：

### 1. 意图分析与匹配
- 首先分析用户的需求（如所需组件类型、色彩倾向、设计风格等）。
- 使用文件系统工具查阅 `.trae/skills/ui-styler/ui_candidates_list.md`，或者检索 `.trae/skills/ui-styler/templates/` 目录中的 `meta.json`。
- 根据特征标签（Tags）、类别（Kind）和描述，筛选出最匹配用户意图的模板目录。

### 2. 提取样板代码
- 定位到匹配的模板目录（例如：`.trae/skills/ui-styler/templates/001-hero-gradient-product`）。
- 读取该目录下的 `index.html` 文件。
- 分析并提取该样板中的 HTML 结构设计、Tailwind CSS 类名、响应式断点（Responsive breakpoints）、暗黑模式处理等关键视觉代码。

### 3. 生成或调整用户代码
- **对于新建场景**：直接基于提取的 Tailwind 类名和结构，生成符合用户所用技术栈（如 React、Vue、原生 HTML 等）的新组件代码。
- **对于修改场景**：在不破坏用户原有业务逻辑的前提下，替换或整合用户的 CSS/类名，将选中样板的设计语言（如阴影层级、圆角大小、色彩搭配、间距规范）应用到现有代码中。

### 4. 反馈结果
- 提交更新后的代码给用户。
- （可选）向用户简短说明采用了哪种风格的样板（如：“为您应用了极简黑白灰风格的卡片设计”），并告知其特点。

## 资源结构
- `preview.html`: 这是一个**包含 100 个模板的高清截图画廊**。用户可以直接在本地浏览器中打开它，通过可视化网格直观地浏览、搜索、并点击预览完整的 UI 设计效果（支持长截图平滑滚动）。这是**给用户展示模板库全貌的最佳入口**。
- `viewer.html`: 这是一个**基于 Tailwind CDN 的动态预览沙箱**。它通过安全的 `postMessage` 与 Blob URL 机制，在隔离的 `iframe` 内完美呈现任何带有 Tailwind Class 的 HTML 代码片段。用户从 `preview.html` 点击“View Full”时，即会跳转到此页面查看完整交互。
- `template-data.js`: 预先打包的 100 个模板的 HTML 字符串字典，供 `viewer.html` 和 `preview.html` 在本地离线（file:// 协议）无跨域报错地快速读取。
- `ui_candidates_list.md`: 100个模板的速查表，包含每个模板的类型、URL来源、设计标签和一句话描述。作为 Agent 筛选风格时的知识库参考。
- `templates/`: 存放所有样板代码及资源的目录，包含从 `001` 到 `100` 的子文件夹。每个子文件夹均有：
  - `index.html`: 核心样板文件，包含所有必要的 Tailwind 类名及 HTML 结构（Agent 读取的重点）。
  - `meta.json`: 该样板的元数据信息（主题、分类、风格标签等）。
  - `preview.png`: 自动生成的该样板的高清渲染截图（供 `preview.html` 画廊使用）。
